Software testing is the process of evaluating software to ensure it meets requirements.

It can be done manually or automatically, and at different levels of detail.

Let's take a look at some common testing metaphors...

Testing Pyramid

The testing pyramid is a metaphor for the different levels of testing.

It shows how we can focus on different types of tests at different levels.

We can use it to guide our testing strategy and priorities.

Testing Onion

The testing onion is a metaphor for the different types of tests.

It shows how we can peel back layers of testing to find defects.

We can use it to understand the different types of tests and their purposes.

OODA Loop

The OODA loop is a decision-making process that stands for Observe, Orient, Decide, and Act.

It was developed by military strategist John Boyd to help make quick and effective decisions.

We can use it to improve our testing practices, and outcomes, and impacts.

OODA loop for testing

Observe

Orient

Decide

Act

Observe: Gather information about the software and its environment.

Orient: Analyze the information and understand the context.

Decide: Make decisions based on the analysis and priorities.

Act: Execute the decisions and take action to improve the software.

Launch Loop

Also known as the continuous integration loop.

It's repeating way to evolve a shared repository.

We're aiming to improve our work throughout the loops.

Launch loop for testing

People first

Delivery is key

Automation may help

People first: We must prioritize the people who use and create the software.

Delivery is key: We must focus on delivering value to our users quickly and reliably.

Automation may help: We can use automation to improve our testing practices and outcomes.

DORA metrics

DORA metrics are a set of four key performance indicators for software delivery.

They help us measure our performance and identify areas for improvement.

Let's take a look at each of them in detail...

DORA metrics

🚀 Launch Frequency

⏱️ Lead Time

⛔️ Defect Rate

✅ Recovery Time

Launch Frequency: How often we deploy code to production.

Lead Time: How long it takes to go from code commit to production.

Defect Rate: How many defects we find in production.

Recovery Time: How long it takes to recover from a production incident.

Selenium Automation
